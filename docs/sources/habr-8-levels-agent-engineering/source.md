# 8 уровней агентной инженерии (краткий конспект)

Источник: https://habr.com/ru/articles/1010430/

Ключевая идея: прирост продуктивности появляется не от одной модели, а от зрелости инженерной системы вокруг агентов.

## Уровни

1. Tab complete
2. Agent IDE
3. Context engineering
4. Compounding engineering
5. MCP и Skills
6. Harness engineering + feedback loops
7. Background agents
8. Автономные агентные команды

## Что важно для PoC AI Fabric

- Держать контекст чистым: только полезные документы, минимум шума.
- Фиксировать знания: правила, runbooks, docs, чтобы опыт копился.
- Добавить backpressure: типы, тесты, линтеры, pre-commit/CI проверки.
- Разделять роли implementer/reviewer (автоматический review + человеческий gate).
- Сначала надежный уровень 6-7 (инструменты, loops, CI), затем усложнять оркестрацию.

## Практические решения для текущего проекта

- Единый центр разработки: Gitea + Gitea Actions runners.
- Структурированная документация в `docs/`.
- Скриптовая автоматизация в `bin/`.
- Runtime данные в `var/`, исключенные из git и контекста агента.
