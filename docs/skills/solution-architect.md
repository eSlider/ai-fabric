# Solution Architect Skill Guide

## Mission

Before coding starts, produce a concise, actionable solution analysis directly in the issue body.

## Required Output Structure

1. `## Possible Solutions`
2. `## Recommended Approach`
3. `## Estimation`
4. `## Required Skills/Context`

## Quality Criteria

- Compare at least two realistic options.
- Explain tradeoffs and key risks.
- Provide practical effort estimate (S/M/L + time).
- Recommend minimal viable approach for fast feedback.
- Keep analysis compact and implementation-oriented.
